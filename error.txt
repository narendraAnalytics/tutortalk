# 🧠 3. Exam Flow (Very Important)

## 🎤 Voice-Based MCQ Flow

### Step-by-step:

1. AI asks question (voice + text)
2. AI reads 4 options
3. User answers via voice:

   * “Option 2” / “B”
4. System:

   * Converts → transcript
   * Maps → option index
5. AI confirms:

   > “You selected Option 2”
6. Evaluate:

   * Correct / Incorrect
7. AI gives feedback
8. Move to next question

# ⚠️ Why combining them is a bad idea

If you merge:

* 🎓 *Exam Conductor Prompt*
* 🧠 *Question Generator Prompt*

You create conflicts like:

* Model tries to **generate + conduct + evaluate** at the same time
* Less consistent difficulty
* Harder to debug
* Unpredictable question quality

👉 In practice, this leads to:

> ❌ inconsistent exams
> ❌ repeated questions
> ❌ wrong difficulty levels

---

# ✅ Correct Architecture (Best Practice)

You should **separate concerns**:

## 🧠 1. Question Generator (Offline / Before Exam)

* Generates questions
* Ensures:

  * syllabus alignment
  * difficulty control
  * clean structure

---

## 🎤 2. Exam Conductor (During Exam)

* Only:

  * asks questions
  * captures answers
  * evaluates

👉 This separation is **very important**

---

# 🧩 Recommended Flow

### Step 1: Generate Questions

* Call LLM with **Question Generator Prompt**
* Store questions (array / DB)

---

### Step 2: Start Exam

* Pass stored questions to **Exam Conductor**
* No generation during exam

---

### Step 3: Conduct Exam

* Ask → Listen → Evaluate → Next

---

# 🧠 Question Generator Prompt (Use Separately)

Here’s a **clean, controlled prompt**:

You are an expert question setter for Class 10 and Intermediate students.

Generate high-quality multiple-choice questions based on the given context.

---

### Context

* Class Level: {level} (Class 10 / Intermediate)
* Subject: {subject}
* Topic: {topic}
* Difficulty: {difficulty} (Easy / Medium / Hard)
* Number of Questions: {n}

---

### Instructions

1. Generate exactly {n} multiple-choice questions.

2. Each question must have:

* A clear question statement
* Exactly 4 options
* Only one correct answer

3. Ensure:

* Questions are syllabus-aligned
* No ambiguity in correct answer
* No repeated questions
* Difficulty matches the requested level

4. Keep language appropriate to the class level:

* Class 10 → simple and clear
* Intermediate → slightly more analytical

---

### Output Format (STRICT JSON)

[
{
"question": "...",
"options": ["...", "...", "...", "..."],
"correct_answer": 1,
"explanation": "..."
}
]
